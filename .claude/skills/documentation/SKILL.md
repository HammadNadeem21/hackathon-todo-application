# Documentation Reviewer Skills

This document outlines the skills and capabilities of the doc-reviewer agent when reviewing and improving technical documentation including specs/*.md, CLAUDE.md, README.md, and other documentation files in the todo application project.

## Core Capabilities

### Documentation Standards & Best Practices
- Applying technical writing principles and style guides
- Ensuring consistency in terminology and formatting
- Following documentation standards (Markdown, etc.)
- Maintaining professional tone and clarity
- Creating accessible documentation for all audiences
- Adhering to project-specific documentation conventions

### Technical Documentation Review
- Reviewing API documentation for accuracy and completeness
- Evaluating code comments and inline documentation
- Assessing architecture and design document quality
- Checking technical accuracy of implementation details
- Validating example code and usage patterns
- Ensuring documentation reflects current codebase

### Spec & Requirements Documentation
- Reviewing feature specifications (specs/*.md) for clarity
- Identifying gaps or ambiguities in requirements
- Ensuring acceptance criteria are testable and complete
- Validating technical feasibility of proposed solutions
- Checking alignment between specs and implementation
- Improving specification structure and organization

### User Documentation
- Reviewing README files for setup and usage instructions
- Evaluating onboarding documentation for new developers
- Assessing API reference documentation quality
- Improving user guides and tutorials
- Creating or enhancing troubleshooting guides
- Validating installation and configuration instructions

### Code Documentation
- Reviewing JSDoc, Python docstrings, and other code comments
- Ensuring proper API documentation generation
- Improving inline documentation for complex logic
- Creating or enhancing example code snippets
- Validating that documentation matches code behavior
- Checking that examples are functional and up-to-date

### Quality Assurance
- Identifying and fixing grammatical errors and typos
- Ensuring logical flow and readability
- Checking for completeness and comprehensiveness
- Validating cross-references and internal links
- Testing example code and commands in documentation
- Ensuring documentation is up-to-date with code changes

### Structure & Organization
- Improving document hierarchy and navigation
- Organizing content for optimal user experience
- Creating effective table of contents and indexes
- Ensuring logical grouping of related information
- Improving searchability and discoverability
- Maintaining consistent document structure

### Technical Accuracy
- Verifying code examples are correct and functional
- Ensuring API endpoints and parameters are accurate
- Validating configuration examples and settings
- Checking that diagrams and visual elements are correct
- Confirming that deployment instructions are current
- Ensuring security recommendations are up-to-date

### Localization & Accessibility
- Improving document accessibility for users with disabilities
- Ensuring documentation supports internationalization
- Creating inclusive language and examples
- Following accessibility guidelines for technical content
- Improving readability for non-native English speakers
- Validating that visual elements have proper descriptions

### Version Control & Maintenance
- Tracking documentation changes and versioning
- Coordinating documentation updates with code changes
- Maintaining changelogs and release notes
- Ensuring documentation follows semantic versioning
- Creating processes for documentation review and approval
- Managing documentation lifecycles and deprecation

## Integration with Todo Application
- Reviewing and improving feature specifications in specs/*.md
- Enhancing project documentation (README.md, CONTRIBUTING.md)
- Maintaining CLAUDE.md and other project-specific docs
- Ensuring API documentation aligns with FastAPI backend
- Improving Next.js frontend documentation
- Validating authentication documentation for Better Auth integration
- Creating and maintaining user guides for todo application features
- Ensuring documentation reflects the monorepo structure and workflows
